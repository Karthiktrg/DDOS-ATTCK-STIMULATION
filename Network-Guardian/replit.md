# Workspace

## Overview

pnpm workspace monorepo using TypeScript. Each package manages its own dependencies.

## Stack

- **Monorepo tool**: pnpm workspaces
- **Node.js version**: 24
- **Package manager**: pnpm
- **TypeScript version**: 5.9
- **API framework**: Express 5
- **Database**: PostgreSQL + Drizzle ORM
- **Validation**: Zod (`zod/v4`), `drizzle-zod`
- **API codegen**: Orval (from OpenAPI spec)
- **Build**: esbuild (CJS bundle)

## Application: AI-Integrated Network Traffic Analysis and DDoS Prevention Tool

### Credentials
- Admin: `admin` / `admin123`
- Analyst: `analyst` / `analyst123`
- Viewer: `viewer` / `viewer123`

### Features
- Real-time network traffic monitoring (live simulation)
- AI-based anomaly detection using Isolation Forest algorithm
- Automatic DDoS attack detection and IP blocking
- Attack history and incident management
- System health monitoring
- CSV export of incidents

## Database Schema

- `users` — Admin/analyst/viewer accounts
- `alerts` — Attack alerts generated by AI engine
- `blocklist` — Blocked IP addresses
- `incidents` — Full attack incident records
- `traffic_history` — Time-series traffic data for charting

## API Routes

All routes are under `/api`:

- `POST /auth/login` — Login, returns JWT token
- `GET /auth/me` — Current user info
- `GET /traffic/snapshot` — Real-time traffic metrics
- `GET /traffic/history` — Historical traffic data
- `GET /traffic/top-ips` — Top IPs by traffic
- `GET /traffic/protocol-distribution` — Protocol breakdown
- `GET /alerts` — Recent alerts (filterable by severity)
- `POST /alerts/:id/acknowledge` — Acknowledge an alert
- `GET /blocklist` — Blocked IP list
- `POST /blocklist` — Manually block an IP
- `DELETE /blocklist/:id` — Unblock an IP
- `GET /incidents` — Attack history
- `GET /incidents/export` — Export CSV
- `GET /stats/dashboard` — Summary stats
- `GET /stats/system-health` — System health metrics

## TypeScript & Composite Projects

Every package extends `tsconfig.base.json` which sets `composite: true`. The root `tsconfig.json` lists all packages as project references. This means:

- **Always typecheck from the root** — run `pnpm run typecheck` (which runs `tsc --build --emitDeclarationOnly`). This builds the full dependency graph so that cross-package imports resolve correctly. Running `tsc` inside a single package will fail if its dependencies haven't been built yet.
- **`emitDeclarationOnly`** — we only emit `.d.ts` files during typecheck; actual JS bundling is handled by esbuild/tsx/vite...etc, not `tsc`.
- **Project references** — when package A depends on package B, A's `tsconfig.json` must list B in its `references` array. `tsc --build` uses this to determine build order and skip up-to-date packages.

## Root Scripts

- `pnpm run build` — runs `typecheck` first, then recursively runs `build` in all packages that define it
- `pnpm run typecheck` — runs `tsc --build --emitDeclarationOnly` using project references
- `pnpm --filter @workspace/api-spec run codegen` — regenerate API client and Zod schemas from OpenAPI spec
- `pnpm --filter @workspace/db run push` — push DB schema changes
